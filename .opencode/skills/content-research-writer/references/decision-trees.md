# Decision Trees: Handling Edge Cases

This guide helps you route to the right approach when common writing scenarios arise.

## When Writer Says: "I'm Stuck, Don't Know How to Continue"

**Diagnosis Flow**:

```
Ask: "What part of the process feels blocked?"

├─ "I don't have an outline"
│  └─ ROUTE: Back to Collaborative Outlining
│     Questions: Topic clear? Audience clear? Structure makes sense?
│
├─ "Outline exists but I'm frozen on a section"
│  └─ ROUTE: Section-Specific Unblocking
│     Questions: Have you written the first draft? What's the section's main point?
│
├─ "I've written everything but it doesn't fit together"
│  └─ ROUTE: Structure & Flow Review
│     Actions: Review section transitions, check logical progression, verify outline still applies
│
└─ "I have research but can't explain it coherently"
   └─ ROUTE: Research Synthesis Help
      Actions: Help distill key points, identify 1-2 main claims, structure evidence
```

**What NOT to Do**: Rewrite the stuck section. Instead, ask diagnostic questions until the writer articulates what's missing.

## When Multiple Research Sources Contradict Each Other

**Evaluation Matrix**:

| Scenario | Example | Decision Rule |
| --- | --- | --- |
| **Same claim, different statistics** | "40% of teams use AI" vs "67% use AI" | Go with most recent, most reputable source. Cite the one you use. Note the disagreement if both are credible. |
| **Different interpretation of same data** | Study shows X, but interpretations differ | Quote the actual data, let writer choose interpretation. Don't assume which is "right." |
| **One source is much newer** | 2019 finding vs 2024 finding on same topic | Prefer recent unless older source is more authoritative or still accurate. Ask writer: "Should we use updated data?" |
| **Sources have clear bias** | Company study vs independent research on same topic | Disclose bias in citation. Use independent source as primary, note company perspective as counterpoint. |
| **Conflict in expert opinion** | Expert A says X, Expert B says Y | Both are valid. Include both perspectives if central to thesis, or ask writer which aligns with their argument. |

**Protocol**:

1. **Don't choose for the writer** — present options with source reputation
2. **Explain the conflict** — "Both are credible, but they emphasize different aspects"
3. **Ask for guidance** — "Which interpretation aligns with your thesis?"
4. **Document the decision** — note why this source was chosen

## When Writer Rejects Your Feedback

**Response Framework**:

```
Step 1: Listen (Don't Defend)
├─ Ask: "What doesn't feel right about the feedback?"
├─ Listen without rebuttal
└─ Understand their reasoning

Step 2: Validate (Even If You Disagree)
├─ "I understand why that doesn't work for you"
├─ Acknowledge their ownership
└─ "It's your voice, your choice"

Step 3: Offer Alternatives
├─ "Here are two other approaches to the same problem..."
├─ OR "Maybe that wasn't the right suggestion; what would help?"
└─ Never push a single solution

Step 4: Move Forward
├─ Accept their choice gracefully
├─ Don't revisit rejected feedback in next section
└─ Adjust your feedback style based on what worked
```

**Key Principle**: Writer's voice > Your suggestion. Always.

## When Writer Wants Feedback on Technical vs Narrative Writing

**Content-Type Routing**:

| Content Type | Feedback Focus | What to Check | What to Skip |
| --- | --- | --- | --- |
| **Technical Documentation** | Clarity, accuracy, completeness | Are steps testable? Is jargon explained? Are edge cases noted? | Emotional hooks, narrative flow, personal voice |
| **Blog Post** | Voice, engagement, pacing | Does hook grab? Are examples relatable? Does tone stay consistent? | Citation formality, exhaustive sources, academic rigor |
| **Research Paper** | Argument strength, evidence, rigor | Are claims supported? Citations complete? Logic sound? | Narrative engagement, conversational tone, personality |
| **Thought Leadership** | Unique POV, credibility, impact | Is the perspective original? Is it well-reasoned? Does it challenge assumptions? | Exhaustive research, neutral tone, balanced coverage |
| **Case Study** | Narrative + data balance | Does story come alive? Are metrics clear? Is the transformation visible? | Academic rigor, exhaustive background, unrelated context |
| **Newsletter** | Brevity, personality, relevance | Is it skimmable? Does voice come through? Is the takeaway clear? | Depth, comprehensive research, detailed citations |

**Before Feedback Ask**: "What type of writing is this? What's your primary goal?"

## When Writer Asks: "Should I Include This?"

**Inclusion Decision Tree**:

```
Question: "Does this serve the main argument?"

├─ YES: "Keep it"
│  └─ Does it support the reader's journey?
│     ├─ YES → Position it strategically
│     └─ NO → Move it to appendix or cut
│
└─ NO: "Consider cutting it"
   ├─ Is it interesting but tangential?
   │  ├─ YES → Offer two options:
   │  │  1. Cut and link to separate piece
   │  │  2. Create a sidebar/callout (doesn't interrupt flow)
   │  └─ NO → Cut
   │
   └─ Is it a lost love (writer wants it despite weak relevance)?
      ├─ YES → Compromise: one sentence reference only
      └─ NO → Cut cleanly and move on
```

**Your Role**: Don't decide. Help writer decide by asking: "Does this move your main argument forward?"

## When Citations Need Consistency Audit

**Citation Format Decision**:

```
Ask writer: "Which citation style do you prefer?"

├─ INLINE: Author (Year) or [link]
│  └─ Best for: Flowing narrative, web-based content, blogs
│
├─ NUMBERED: Superscript[1] with footnotes at end
│  └─ Best for: Academic papers, long-form articles, complex citations
│
└─ FOOTNOTE: Endnote style with full citations
   └─ Best for: Technical docs, formal reports, scholarly work
```

**Then**:

- Audit entire document for consistency
- Convert any mismatched citations to chosen style
- Create a references list using that style
- Flag any citations missing publication details

**Red Flag**: If different sections use different styles, that's a professionalism issue. Fix it.

## When Writer Is Overwhelmed by Feedback

**Triage & Prioritize**:

```
Identify: What type of feedback is most important?

1. CRITICAL (must fix):
   ├─ Factual errors or unsourced claims
   ├─ Logical gaps in argument
   └─ Missing citations that you flagged

2. IMPORTANT (should improve):
   ├─ Clarity issues affecting understanding
   ├─ Flow/structure problems
   └─ Weak transitions

3. OPTIONAL (nice to have):
   ├─ Word choice alternatives
   ├─ Style tweaks
   └─ Nice-to-have examples
```

**Then**: Present only CRITICAL + 2-3 IMPORTANT items. Save OPTIONAL for next pass.

**Say**: "Here are the 3 things that will have biggest impact. Ready to tackle these?"

## When Writer Needs to Adapt Content for Different Audiences

**Audience Adaptation Framework**:

| Audience | Adjust | Keep | Example |
| --- | --- | --- | --- |
| **From Technical → Business** | Jargon removed, ROI framed, examples simplified | Core insights, data, credibility | Replace "ML algorithms" with "automated decision-making" |
| **From Narrative → Academic** | Anecdotes reduced, formal citations required, neutral tone | Core argument, evidence, originality | Add literature review, formal citations, remove storytelling |
| **From Long-form → Social Media** | Length cut 90%, snappier sentences, hook at start | Main takeaway, credibility, call-to-action | Thread 10 posts from 2000-word article |
| **From Expert → Beginner** | Jargon explained, more examples, foundational context | Nuance, credibility, depth | Add "What is X?" section before diving into analysis |

**Your Role**: Identify what changes, what stays. Let writer decide adaptation approach.

---

**Summary**: These decision trees help you diagnose situations quickly and route to the right approach. When stuck, ask diagnostic questions before offering solutions.
