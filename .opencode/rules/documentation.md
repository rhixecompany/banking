# Documentation Standards

- Keep README.md concise and focused on developer onboarding.
- Document complex logic in-code with TSDoc/TSDoc-appropriate comments.
- Use consistent heading hierarchy and avoid deep nesting in markdown.
- Store detailed docs in `/docs` subdirectory; keep project root clean.
- Link to external resources (API docs, third-party services) rather than duplicating.
- Update docs in the same commit as the code they describe.
